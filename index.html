Ink is a simple scripting language used to create interactive stories ‚Äî like Choose Your Own Adventure games. It powers narrative games such as 80 Days and Heaven‚Äôs Vault.

Ink scripts are built from knots (story sections) connected by diverts (links). Players make choices that determine what happens next.

2. Understanding the Basics

  • Knots: Sections of the story, written as === name ===.
  • Diverts: Use -> name to jump to another knot.
  • Choices: Use + to present an option to the player.
  • END: Marks the end of a story path.
Think of knots like paragraphs, diverts like hyperlinks, and choices like buttons the player can press.

3. Variables and Logic

Ink can remember things! You can store numbers and true/false values in variables.

  • Integer variable: Stores a number, e.g. VAR lives = 3
  • Boolean variable: Stores true or false, e.g. VAR has_key = false

You can use conditional text blocks to make the story react dynamically:

{lives > 0:
    You have {lives} lives left.
- else:
    You‚Äôre out of lives!
}

The first part runs if the condition is true, otherwise Ink shows the else block.

5. How Ink ‚ÄúThinks‚Äù

  • When Ink sees {lives}, it substitutes the current number.
  • When it reaches ~ lives = lives - 1, it updates the stored value.
  • When it reads {condition: ... - else: ...}, it checks which part to display.

Ink keeps variable values in memory for the rest of the story (unless you reset them).
